Serving Videos >25 MB on Cloudflare Pages

Status: Pending approval Date: 2026-05-19 Author: technical-research-agent Repo: playwright-typescript

📋 1. Summary

⚠️ Cloudflare Pages enforces a hard 25 MiB per-file limit — no override exists on any plan.

The current CI workflows (lighthouse-ci.yml lines 114–117, playwright-with-slack.yml lines 426–428 and 444–446) already include find ... -size +25M -delete steps that silently drop oversized files before each Cloudflare Pages deploy. Large Playwright failure videos are lost today.

The fix is to offload files >25 MiB to a separate store and rewrite the HTML report links to point there. The recommended store is Cloudflare R2 (same vendor, same credentials, zero egress fees, 10 GB free tier).

✅ 4. Recommendation

Option A — Cloudflare R2 with a public bucket (custom domain optional but advised)
  • Reuses existing CLOUDFLARE_API_TOKEN / CLOUDFLARE_ACCOUNT_ID secrets and wrangler tooling already in CI.
  • Zero egress fees. 10 GB free tier far exceeds typical test-video volume.
  • Same vendor/outage domain as Pages — one dashboard, one status page.
  • Directly replaces silent-delete behaviour with "offload + link" — silently-lost videos become viewable.
ℹ️ Conditions before implementing:
1. Widen the API token to include R2 Storage: Edit permission.
2. Confirm whether failure videos are sensitive (show app data/UI) — if yes, upgrade to Option B (presigned URLs).
3. Add an R2 lifecycle rule to cap unbounded bucket growth.

🔧 6. Implementation Steps

For the technical-implementation-agent — only after user approval.

  1. Create R2 bucket (e.g. playwright-report-media) in Cloudflare dashboard; enable public access via r2.dev or bind a custom domain.
  2. Widen API token — update the existing CLOUDFLARE_API_TOKEN secret (or add a dedicated R2_API_TOKEN) to include R2 Storage: Edit permission.
  3. Add offload script scripts/offload-large-files-to-r2.js (mirrors scripts/generate-lhci-index.js): finds files >25 MiB in the report directory, uploads them to R2 under a run-scoped prefix (${run_id}/...), rewrites matching src/href references in report HTML/JSON to R2 public URLs.
  4. Update CI workflows:
    — lighthouse-ci.yml lines 114–117
    — playwright-with-slack.yml lines 426–428 and 444–446
    Replace find ... -size +25M -delete with a call to the offload script. Keep continue-on-error: true.
  5. Add R2 lifecycle rule (or scheduled CI prune job) to expire media after 30–90 days to bound storage growth.
  6. (Optional) Surface the R2 base URL as a workflow output / Slack notification field for traceability.

🧪 7. Validation

✔️ All four checks must pass before closing the implementation task.
