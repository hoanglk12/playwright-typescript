📋 Summary

The current UI lifecycle (src/config/global-setup.ts, src/config/global-teardown.ts) works but carries several concrete correctness gaps:

  • No fail-fast on connectivity — a down environment only console.warns; the full suite runs and fails downstream instead of aborting.
  • Connectivity checks miss the real targets — only adminUrl/frontSiteUrl are pinged; the 8 Accent storefronts that ecommerce specs depend on aren't gated by setup at all.
  • Wasteful browser validation — launches Chromium, Firefox, and WebKit every run, even though WebKit's project is commented out and Firefox is excluded from ecommerce in CI.
  • archiveTestArtifacts() is a silent no-op — a fs.statSync(...).isFile() guard means report directories are never archived; only test-summary.txt gets copied, and CI's upload-artifact already covers everything real.
  • No storageState auth reuse, no required-secrets validation, hardcoded 15000 timeout instead of a TIMEOUTS constant, unused imports (path, exec/execAsync/os).

✅ Recommended Approach — Direction A (Approved)

Harden the existing two files first: the highest-value problems (no fail-fast, wrong connectivity targets, redundant browser launches, dead archive function) are all fixable inline with a small, low-risk, easily reversible change — no restructure of playwright.config.ts and no impact on the 6-shard CI matrix or the monocart trend pipeline.

Direction B (project-dependencies + storageState auth caching) remains conditional on a future count of login-heavy specs and is out of scope for this implementation pass.
