⚠️

Free tier reality

Only NopeCHA has a recurring free quota (100/day), but its primary integration is a browser extension — incompatible with headless CI and this framework's multi-browser Playwright config. One parallel CI run exhausts the 100/day limit. No free option is viable for CI-scale use. Use always-pass test sitekeys in staging instead.

Options

Option A — Test-mode keys in staging

Configure the SUT's staging environment with always-pass CAPTCHA sitekeys. Zero solver integration needed, zero cost, zero CI latency.

✓ Zero cost · zero new deps · deterministic
✗ Requires site-team config access · no staging = blocked

Option B — 2captcha + CaptchaHelper ✓ Recommended

New CaptchaHelper fixture (PercyHelper pattern) wired to @2captcha/captcha-solver. Multi-type: reCAPTCHA v2/v3, Turnstile, hCaptcha. Env-gated; off by default in CI.

✓ Official TS SDK · broadest coverage · clean fit into POM
✗ Ongoing cost · 10–60s solver latency per test

Option C — NopeCHA free tier

Browser extension approach. Free for ≤100 req/day. Conflicts with headless CI launchOptions; one CI run exhausts the quota.

✓ Free for local manual use
✗ Extension conflicts with headless · 100/day is insufficient for CI

Option D — Datadome bypass stack (rejected)

Stealth plugin + residential proxy + paid Datadome task. High cost, fragile, breaks Playwright tracing and Percy. ToS violation on third-party sites.

✗ Not appropriate for a QA framework — out of scope
3
Architecture Decision
🏗️

Fixture-only pattern — mirror PercyHelper, do NOT promote to BasePage

CAPTCHA solving is opt-in per test. Adding it to BasePage would instantiate the helper (and load env vars) for every test, including those that never encounter a CAPTCHA. The percyHelper fixture pattern already solves this — apply it exactly the same way.

// src/config/base-test.ts — add alongside percyHelper
captchaHelper: async ({ page }, use) => {
  const helper = new CaptchaHelper(page);
  await use(helper);
},

// Usage in a test spec
test('TC_01 - Solve Turnstile on checkout', async ({ captchaHelper, page }) => {
  const token = await captchaHelper.solveTurnstile(sitekey, pageUrl);
  await captchaHelper.injectToken(token);
});

Implementation Steps

Install dependency

Run npm install @2captcha/captcha-solver --save-exact. Pin exact version to prevent drift.

Create src/data/captcha/captcha-test-data.ts

Typed interface CaptchaTestKeys + const CAPTCHA_TEST_KEYS with all always-pass sitekeys from the table above. Follow the AdminData reference pattern from CLAUDE.md.

Create src/pages/helpers/captcha-helper.ts

  • Constructor takes page: Page (same as all other helpers)
  • Reads process.env.CAPTCHA_ENABLED, CAPTCHA_API_KEY, CAPTCHA_PROVIDER
  • Methods: solveTurnstile, solveRecaptchaV2, solveRecaptchaV3, solveHcaptcha
  • injectToken(token, targetSelector?) — page.evaluate to set hidden input value
  • All throw clearly if CAPTCHA_ENABLED !== 'true'
  • Use TIMEOUTS.CAPTCHA_SOLVE for solver timeout
  • All locators declared as private readonly class fields per CLAUDE.md

Export from src/pages/helpers/index.ts

Add export { CaptchaHelper } from './captcha-helper';

Add CAPTCHA_SOLVE to src/constants/timeouts.ts

CAPTCHA_SOLVE: process.env.CI ? 180000 : 120000

Wire fixture in src/config/base-test.ts

Mirror the percyHelper fixture exactly. Add captchaHelper: CaptchaHelper to CustomFixtures type.

Create tests/captcha/captcha-helper-smoke.spec.ts

  • test.skip(!process.env.CAPTCHA_ENABLED, 'CAPTCHA_ENABLED not set') guard
  • TC_01: Turnstile demo with always-pass test sitekey → verify "Success" state (no credits)
  • TC_02: reCAPTCHA v2 demo with Google test sitekey → verify token injected (no credits)
  • TC_03 (optional): 2captcha demo page → real credits, manual smoke only
  • test.skip(browserName === 'firefox', '...') guard (Juggler teardown workaround)

Update .env.* files

Add three new vars to .env.testing, .env.staging, .env.production:
CAPTCHA_API_KEY=  ·  CAPTCHA_ENABLED=false  ·  CAPTCHA_PROVIDER=2captcha

Update CI workflows

Add CAPTCHA_API_KEY: ${{ secrets.CAPTCHA_API_KEY }} env block to playwright.yml and playwright-with-slack.yml. No-op when CAPTCHA_ENABLED=false. Do not touch API workflow files.

Update CLAUDE.md

Add row to fixture table: captchaHelper | CaptchaHelper | captcha solving

Lint check

Run npm run lint — must pass with zero TypeScript errors.
