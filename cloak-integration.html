📋 Context

Standard Playwright Chromium leaks automation signals — CDP flags, canvas fingerprint, WebGL, audio context, hardware emulation — causing bot detectors to block or challenge it. CloakBrowser is a custom Chromium build with 58 source-level C++ stealth patches that make the browser indistinguishable from human-driven Chrome.

Scope rule: Only the one test scenario that hits a CAPTCHA-protected flow uses CloakBrowser. All other specs, Firefox runs, and API tests remain completely untouched.
Note: CloakBrowser prevents CAPTCHAs from being triggered — it does not solve existing ones. If CAPTCHA can be disabled in your QA/staging environment (via Google's test site keys), that is the cleaner first option. Use CloakBrowser for staging/production smoke tests where CAPTCHA cannot be turned off.

🏗️ Architecture

Create a self-contained cloakPage fixture that spins up a CloakBrowser instance, creates a page, and tears everything down after the test. The one test scenario that needs CAPTCHA bypass imports this fixture. No changes to base-test.ts, playwright.config.ts, or any other test file.

1 · package.json — add optional dependency

// optionalDependencies so CI doesn't break if it can't install
"optionalDependencies": {
  "cloakbrowser": "latest"
}

2 · src/config/cloak-test.ts — new fixture file

src/config/cloak-test.ts 
import { test as base, expect } from '@playwright/test';
import { Browser, BrowserContext, Page } from '@playwright/test';

type CloakFixtures = {
  cloakBrowser: Browser;
  cloakContext: BrowserContext;
  cloakPage: Page;
};

export const test = base.extend<CloakFixtures>({
  cloakBrowser: [async ({}, use) => {
    let browser: Browser;
    try {
      const { launch } = await import('cloakbrowser');
      browser = await launch({
        headless: process.env.HEADLESS !== 'false'
      }) as unknown as Browser;
    } catch {
      throw new Error(
        'CloakBrowser not installed. Run: npm install cloakbrowser\n' +
        'Required for CAPTCHA-bypass tests.'
      );
    }
    await use(browser);
    await browser.close();
  }, { scope: 'test' }],

  cloakContext: async ({ cloakBrowser }, use) => {
    const context = await cloakBrowser.newContext();
    await use(context);
    await context.close();
  },

  cloakPage: async ({ cloakContext }, use) => {
    const page = await cloakContext.newPage();
    await use(page);
    await page.close();
  },
});

export { expect };

3 · The single CAPTCHA test — import swap only

tests/<area>/your-captcha-test.spec.ts 
// Change this one import line in the affected spec file:
// Before:
import { test, expect } from '@config/base-test';

// After:
import { test, expect } from '@config/cloak-test';

// Use cloakPage instead of page in that one test:
test('TC_XX - Description @smoke', async ({ cloakPage }) => {
  await cloakPage.goto('https://protected-site.com/flow');
  // standard Playwright API — identical to page.* calls
});
Need page objects too? If the CAPTCHA test also uses fixtures from base-test.ts (e.g. homePage), extend base-test instead of @playwright/test at the top of cloak-test.ts: import { test as base } from '@config/base-test'.

⚠️ Known Constraints & Risks

RiskSeverityMitigation
TypeScript type mismatch between cloakbrowser and Playwright types LOW as unknown as Browser cast — APIs are identical
Binary not installed in CI MEDIUM optionalDependencies + clear error message; CAPTCHA test fails fast
~200 MB binary download on first run LOW Cache node_modules/cloakbrowser/ via actions/cache
New browser per test adds ~1–2s overhead LOW Acceptable for single scenario; switch to scope: 'worker' if needed
Binary download blocked in restricted network MEDIUM Pre-download locally; use launch({ executablePath: process.env.CLOAKBROWSER_PATH })
CloakBrowser is Chromium only — no Firefox LOW Firefox tests continue using standard Playwright, unaffected

✅ Verification Steps

  • 1 Run npm install — confirm cloakbrowser installs and binary auto-downloads (~200 MB first time)
  • 2 Run the CAPTCHA test in isolation: npx playwright test --grep "TC_XX" — confirm it passes without triggering a CAPTCHA or bot block
  • 3 Run the full suite normally — confirm all other tests pass unchanged and no new failures introduced
  • 4 Run npm run lint — confirm zero TypeScript errors in cloak-test.ts and the modified spec file
  • 5 Check test trace or logs — CloakBrowser Chromium version should appear, distinct from standard Playwright Chromium
