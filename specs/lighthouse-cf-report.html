Lighthouse CI Auto-Trigger + Cloudflare Pages Report Upload

Implementation Plan — Playwright TypeScript Framework
CI/CD Change Cloudflare Pages @lhci/cli ^0.15.0

Context & Motivation

Problem 1 — Trigger gap: lighthouse-ci.yml only fires when specific paths change (src/data/ecommerce/**, lighthouserc.js, the workflow file). Routine code pushes never kick off a Lighthouse audit. The goal is to align it with playwright-with-slack.yml, which fires on every push to main / master / develop.

Problem 2 — No public report URL: Lighthouse reports are only stored as GitHub artifacts (30-day retention, no browsable URL). The goal is to deploy all 8 site reports to a single stable Cloudflare Pages URL after every run — consistent with how UI/API reports are already hosted.

Discovered gap: lighthouserc.js only lists 7 URLs. Dr. Martens NZ (stag-drmartens-nz.accentgra.com) is defined in storefronts.ts but missing from the LHCI config. Adding it delivers the expected 8-site coverage.

Confirmed Decisions
  • Add Dr. Martens NZ as the 8th site in lighthouserc.js
  • Push-only trigger — no pull_request event (avoids doubling 25-min runs per PR)
  • Cloudflare Pages project name: playwright-lighthouse-reports
  • Landing page links to median/representative run only (8 links total)

scripts/generate-lhci-index.js — New landing-page generator

Plain Node.js (no new npm deps). Reads lighthouse-reports/manifest.json, picks the isRepresentativeRun === true entry per URL (falls back to first run if none flagged), derives a human-readable label from the hostname, and writes lighthouse-reports/index.html with 8 links plus commit metadata.

// Pseudocode logic:
const manifest = JSON.parse(fs.readFileSync('lighthouse-reports/manifest.json'));
const byUrl = groupBy(manifest, 'url');
const rows = Object.entries(byUrl).map(([url, runs]) => {
  const rep = runs.find(r => r.isRepresentativeRun) ?? runs[0];
  const label = labelFromUrl(url);   // e.g. 'Platypus AU', 'Dr. Martens NZ'
  return { label, url, htmlPath: path.basename(rep.htmlPath), scores: rep.summary };
});
// Write index.html with <ul> of 8 <a href="..."> links + scores
// Include: branch, sha, timestamp in <h1>

lighthouse-ci.yml — Add landing-page generation step

Insert after the existing lhci:upload step. Passes commit metadata as env vars for the page heading.

  if: always()
  run: node scripts/generate-lhci-index.js
  env:
    GITHUB_SHA: ${{ github.sha }}
    GITHUB_REF_NAME: ${{ github.ref_name }}
7

lighthouse-ci.yml — Add Cloudflare Pages deploy step

Reuses the exact wrangler pattern from playwright-with-slack.yml. No new secrets required — CLOUDFLARE_API_TOKEN and CLOUDFLARE_ACCOUNT_ID already exist at repo level. The project is auto-created by wrangler on first run.

- name: Delete large files before Cloudflare deploy
  if: always()
  run: find lighthouse-reports -type f -size +25M -delete
  continue-on-error: true

- name: Deploy Lighthouse reports to Cloudflare Pages
  if: always() && hashFiles('lighthouse-reports/index.html') != ''
  id: cf-lighthouse
  continue-on-error: true
  run: |
    npx wrangler@3 pages deploy lighthouse-reports \
      --project-name playwright-lighthouse-reports \
      --branch main \
      --commit-message "Lighthouse ${{ github.ref_name }} run ${{ github.run_id }}"
  env:
    CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
    CLOUDFLARE_ACCOUNT_ID: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
